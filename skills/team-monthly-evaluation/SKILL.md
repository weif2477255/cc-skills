# 团队月度评价

自动化生成专业的团队月度综合评价，包括数据清洗、统计分析、绩效评估和评价报告生成。

## 核心能力

1. **数据清洗与统计**：使用Python脚本处理原始txt工作日志，去除无效信息，统计工作量和关键词
2. **模板生成**：自动生成包含完整结构的月度评价模板，减少上下文消耗
3. **绩效评估**：基于四维度评分标准和统计数据提供客观的AI评分建议
4. **评价报告生成**：生成包含团队整体评分、成员评分、优秀事迹、改进建议的专业评价报告

## 使用方式

用户提供txt格式的团队工作日志，要求生成月度评价：

```
请帮我生成2026年1月的团队月度评价，工作日志文件是 work-log-2026-01.txt
```

或者：

```
分析一下团队这个月的工作表现，并给出绩效评分建议
```

## 工作流程

> **⚠️ 重要提示**：
> - **必须先运行清洗脚本**，不能跳过这一步
> - **不要直接读取原始txt文件**，必须通过清洗脚本生成JSON文件后再读取
> - **JSON文件只读取一次**（步骤4），后续步骤5-9都基于已读取的内容进行
> - 清洗脚本会自动完成数据清洗、统计分析和评分参考数据生成

### 步骤1：日志清洗（必须先执行）

运行清洗脚本处理原始txt工作日志，输出JSON格式的清洗后数据和评分参考数据：

```bash
python scripts/clean_work_log.py <input.txt> <output.json>
```

脚本输出JSON文件（`[yyyy-mm]_工作日志.json`），包含summary（统计）、details（工作内容）、scoring_reference（评分参考数据）。

### 步骤2：生成评价模板

运行init_report.py生成包含完整结构的月度评价模板：

```bash
python scripts/init_report.py <input.json> <output.md>
```

脚本会生成包含完整结构的Markdown模板（`[yyyy-mm]团队月度评价.md`），使用TODO标记指示需要填充的部分。这样可以避免从零开始写整个文件，减少上下文消耗。

### 步骤3：读取组织结构

**强制读取** [references/team-structure.md](references/team-structure.md)，了解团队架构和成员岗位信息。

**目的**：评价前知晓员工工作岗位（如项目经理、架构师、前端/后端开发、测试等），评价会更加聚焦和准确。

### 步骤4：读取JSON文件

**一次性读取**清洗后的JSON文件，加载所有数据到上下文：
- `summary`：月份、人员数、工作日数等统计信息
- `details`：每个成员的具体工作内容
- `scoring_reference`：每个成员的评分参考数据

**⚠️ 重要**：此步骤后，步骤5-9都基于已读取的JSON内容进行，**禁止重复读取JSON文件**。

### 步骤5：完成团队整体评价

基于已读取的JSON数据，分析团队整体表现，使用Edit工具填充模板的"一、团队整体评分"部分。

**输出要求**：参考 [references/output-patterns.md](references/output-patterns.md) 第1.2节"团队整体评分"

**⚠️ 重要**：填充完成后，立即清除该部分的所有 `<!-- -->` 注释

### 步骤6：完成团队成员评分

**强制读取** [references/scoring-guide.md](references/scoring-guide.md)，严格按照评分标准为每个成员评分。

基于已读取的JSON数据和评分标准，使用Edit工具填充模板的"二、团队成员评分"部分。

**评分流程**：
1. 阅读每个成员的具体工作内容（`details`字段）
2. 参考评分参考数据（`scoring_reference`字段）
3. 综合判断进行四维度评分（JSON数据是参考，不是最终答案）
4. 确定评级（S/A+/A/B+/B/C），严格按照评级分布规则

**输出要求**：参考 [references/output-patterns.md](references/output-patterns.md) 第1.3节"团队成员评分"

**⚠️ 重要**：
- **表格必须按总分从高到低排序**（这是强制要求，不可忽略）
- 只输出评分表格，不输出详细的评分依据
- 评分分析在内部完成，不输出到报告中
- **填充完成后，立即清除该部分的所有 `<!-- -->` 注释**

### 步骤7：完成优秀成员事迹

基于已读取的JSON数据和团队结构信息（[references/team-structure.md](references/team-structure.md)），使用Edit工具填充模板的"三、优秀成员事迹（S、A+、A评级）"部分。

**输出要求**：参考 [references/output-patterns.md](references/output-patterns.md) 第1.4节"优秀成员事迹"

**⚠️ 重要**：填充完成后，立即清除该部分的所有 `<!-- -->` 注释

### 步骤8：其他成员综合评价

基于已读取的JSON数据，使用Edit工具填充模板的"四、其他成员综合评价"部分。

**输出要求**：参考 [references/output-patterns.md](references/output-patterns.md) 第1.5节"其他成员综合评价"

**⚠️ 重要**：填充完成后，立即清除该部分的所有 `<!-- -->` 注释

### 步骤9：团队改进建议

基于已读取的JSON数据和整体分析，使用Edit工具填充模板的"五、团队改进建议"部分。

**输出要求**：参考 [references/output-patterns.md](references/output-patterns.md) 第1.6节"团队改进建议"

**⚠️ 重要**：填充完成后，立即清除该部分的所有 `<!-- -->` 注释

### 步骤10：质量检查

完成报告生成后，必须执行质量检查，确保报告满足所有要求。

**检查流程**：

1. **读取生成的报告**：使用Read工具读取生成的月度评价报告
2. **读取质量检查清单**：参考 [references/output-patterns.md](references/output-patterns.md) 第三节"质量检查清单"
3. **逐项检查**：按照检查清单的6个类别（结构完整性、数据准确性、评分公平性、格式规范性、内容质量、语言专业性）逐项验证
4. **修复问题**：如果发现问题，使用Edit工具修复
5. **输出检查结果**：向用户报告检查结果，说明是否通过检查，如有问题已修复

**检查要点**：

- **结构完整性**：五个部分齐全，标题层级正确，注释已清除
- **数据准确性**：总分计算正确，排序正确，评级一致
- **评分公平性**：评级分布符合规则（S≤10%、A+≤10%、A≤10%），评分基于数据
- **格式规范性**：表格格式正确，标题层级正确
- **内容质量**：突出贡献有具体事实，综合评价根据岗位撰写，改进建议具体可行
- **语言专业性**：正式、客观、专业，无口语化表达

**⚠️ 重要**：质量检查是强制步骤，不可跳过。如果发现问题，必须修复后再向用户交付报告。
