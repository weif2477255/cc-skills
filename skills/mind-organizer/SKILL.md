# Mind Organizer

整理碎片化想法，转化为结构化任务。

## 核心能力

1. **思维收集与整理** - 接收口语化想法，转化为专业表达，按四维度分类
2. **智能分析** - 运用系统思维、发散思维、第一性原理进行深度分析
3. **可行性评估** - 基于工作职责和产品定位评估优先级
4. **任务规划** - 将高优先级想法转化为可执行任务

## 工作流程

### 步骤1：加载上下文

首先读取 [references/context.md](references/context.md) 了解用户的工作背景、团队情况和评估标准。这是所有分析的基础锚点。

### 步骤2：接收用户输入

接收用户自由表达的想法、灵感、问题。可以是：
- 口语化、碎片化的表达
- 多个不相关的想法
- 模糊的灵感或疑问

### 步骤3：深度分析

对每个想法运用多种思维模型：

- **系统思维**：分析全局关联和影响
- **发散思维**：探索替代方案和创新可能
- **第一性原理**：回归本质问题
- **头脑风暴**：激发更多可能性

**关键**：所有分析必须基于 context.md 中的思维锚点（工作职责、产品定位、团队状态）。

### 步骤4：访谈澄清（按需触发）

当发现以下情况时，使用 AskUserQuestion 工具：

- **不清晰**：想法表达模糊，无法判断具体要做什么
- **有歧义**：存在多种理解方式，需要确认真实意图
- **有更好建议**：基于思维模型分析，发现更优的替代方案
- **优先级冲突**：多个想法都很重要，需要用户权衡取舍

**提问原则**：
- 问题要深入，不问显而易见的问题
- 问题要具体，避免过于宽泛
- 问题要关键，聚焦核心决策点
- 轮次要适度，1-2轮即可

### 步骤5：输出文档

使用 [references/output-template.md](references/output-template.md) 中的模板生成最终文档，包含：

1. **思维整理总览** - 按四维度分类的想法列表（专业化表达）
2. **可行性分析** - 每个想法的分析结论、核心理由、优先级建议、关键风险
3. **任务清单** - P0/P1/P2 分级的可执行任务

## 关键原则

1. **思维锚点优先** - 所有分析必须基于工作职责和产品定位
2. **做减法而非加法** - 帮助用户聚焦，理性放弃不适合当下的想法
3. **访谈按需触发** - 只在不清晰、有歧义、有更好建议时才提问
4. **简洁实用** - 不罗列思维过程，只给结论和理由
5. **粗粒度任务** - 任务拆解到"开发XX功能"级别即可

## 质量检查

输出文档前自问：

- [ ] 是否基于思维锚点（工作职责+产品定位）进行分析？
- [ ] 是否运用了系统思维、发散思维、第一性原理？
- [ ] 是否将口语化表达转为专业表达？
- [ ] 是否给出了清晰的可行性结论和优先级建议？
- [ ] 是否主动指出了关键风险？
- [ ] 任务是否是粗粒度、可执行的？
- [ ] 是否帮助用户做了减法（聚焦而非分散）？
